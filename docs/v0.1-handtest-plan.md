# ACT v0.1 — internal hand-test plan

**Owner:** Jeremy Forsythe (BDFL)
**Date opened:** 2026-05-02
**Status:** Active — gate to v0.2 public release.

This is the soak-window checklist for the v0.1 internal candidate. Every package and example must pass a hand-test pass before v0.2 ships publicly. As issues surface, fix them on `master` and re-run the relevant package's gate; queue any spec-shaped fixes under `docs/amendments-queue.md`.

## How to use this checklist

For each package: smoke-test the public API in a fresh consumer harness, exercise the conformance gate, and read the README from a first-time-user perspective. For each example: build clean from a clean checkout, validate against `@act-spec/validator`, and walk the output by hand.

The harness recipes below assume `pnpm` 10.17.1+ and Node 20+. Use `pnpm pack` from the package directory to produce a tarball and install it into a separate sandbox project to catch packaging bugs that workspace links hide.

Mark each row with date + initials + brief note when checked off. Any item that doesn't pass becomes a `master` fix, not a v0.1 reframing.
